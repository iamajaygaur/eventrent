# Step-by-step: Set up Supabase and connect EventRent

Follow these steps to store rental requests in Supabase instead of the browser’s localStorage.

---

## Step 1: Create a Supabase account and project

1. Go to **[supabase.com](https://supabase.com)** and click **Start your project**.
2. Sign in with **GitHub**, **Google**, or **Email** (choose one and complete sign-up if needed).
3. Click **New project**.
4. Fill in:
   - **Name**: e.g. `eventrent`
   - **Database password**: Choose a strong password and **save it** (you need it for direct DB access; the app uses the anon key, not this password).
   - **Region**: Pick the one closest to you.
5. Click **Create new project** and wait 1–2 minutes until the project is ready.

---

## Step 2: Get your project URL and anon key

1. In the left sidebar, click the **Settings** (gear) icon.
2. Click **API** under “Project settings”.
3. On the API page you’ll see:
   - **Project URL** (e.g. `https://abcdefghijk.supabase.co`)
   - **Project API keys**:
     - **anon public** – use this in the app (safe to use in the browser).
     - **service_role** – do **not** use in the frontend; keep it secret.
4. Copy and save:
   - **Project URL**
   - **anon public** key  
   You’ll paste these into `.env` in Step 5.

---

## Step 3: Create the `rental_requests` table

1. In the left sidebar, click **SQL Editor**.
2. Click **New query**.
3. Paste this entire SQL and click **Run** (or press Cmd/Ctrl + Enter):

```sql
-- Table for rental request submissions
create table public.rental_requests (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now() not null,
  order_id text unique,
  total_price text,
  items jsonb default '[]'::jsonb,
  full_name text,
  department text,
  speedtype text,
  email text,
  phone text,
  address text,
  notes text,
  start_date text,
  start_time text,
  end_date text,
  end_time text
);

-- Row Level Security (RLS): allow the app to insert and read
alter table public.rental_requests enable row level security;

create policy "Allow anonymous insert"
  on public.rental_requests for insert to anon with check (true);

create policy "Allow anonymous select"
  on public.rental_requests for select to anon using (true);
```

4. You should see **Success. No rows returned.**  
5. Check the table: in the left sidebar open **Table Editor** → you should see **rental_requests** with columns including `order_id`, `id`, `created_at`, `total_price`, `items`, etc.

**If you already have the table without `order_id`**, run this in SQL Editor to add it:

```sql
alter table public.rental_requests add column if not exists order_id text unique;
```

---

## Step 4: Create your `.env` file in the project

1. Open your **EventRent** project folder in the terminal.
2. Copy the example env file:

   ```bash
   cp .env.example .env
   ```

3. Open the `.env` file in your editor (it’s in the project root, same folder as `package.json`).

---

## Step 5: Add your Supabase URL and anon key to `.env`

1. In `.env`, replace the placeholders with the values from **Step 2**:

   ```env
   VITE_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-public-key-here
   ```

   Example (with fake values):

   ```env
   VITE_SUPABASE_URL=https://abcdefghijk.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

2. Save the file.  
3. Make sure `.env` is **not** committed to git (it should already be in `.gitignore`).

---

## Step 6: Restart the dev server and test

1. If the app is running, stop it (**Ctrl+C** in the terminal).
2. Start it again:

   ```bash
   npm run dev
   ```

3. Open **http://localhost:5173/**.
4. Submit a **test rental request** (fill items, dates, contact, then click **Submit Rental Request**).
5. In Supabase: open **Table Editor** → **rental_requests**. You should see one new row with your test data.
6. Log in as admin: go to **http://localhost:5173/admin**, sign in, then open **Rental Requests**. The table should show the request you just submitted (from the database). Use **Refresh** to reload.

## Troubleshooting

| Issue | What to do |
|--------|------------|
| “Could not save your request” on submit | Check `.env`: `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` must be set and correct. Restart `npm run dev` after changing `.env`. |
| No rows in Table Editor after submit | In Supabase SQL Editor, run: `select * from public.rental_requests;` to confirm. If still empty, open the browser dev tools (F12) → Console and look for Supabase errors. |
| Rental Requests page empty (but Supabase has rows) | Confirm the same `.env` is used (Vite loads `VITE_*` at build/dev). Restart dev server and hard-refresh the page (Ctrl+Shift+R or Cmd+Shift+R). |
| RLS errors in Supabase logs | Ensure both policies were created (anonymous insert and select). Re-run the full SQL from Step 3. |

- [ ] SQL run in **SQL Editor** (table + RLS policies)
